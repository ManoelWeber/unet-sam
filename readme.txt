# Cutting Tool Wear Segmentation using SAM and U-Net

This repository contains the main scripts used for cutting tool wear segmentation in images of milling inserts. The proposed workflow combines the Segment Anything Model (SAM) and a U-Net-based model to identify and segment flank wear regions in cutting tools.

The pipeline was developed to support the automatic generation of segmentation masks and to reduce the need for manual prompt definition during the use of SAM.

***
Disclaimer: Generative AI tools were used as support during the development and refinement of parts of the code.

## Repository Description

The repository includes three main scripts:

* masc_sam.py: generates initial segmentation masks using SAM from manually defined bounding boxes;
* train_unet.py: trains a U-Net model using the masks generated in the previous step;
* unet_sam.py: applies the hybrid U-Net + SAM pipeline, where U-Net generates a preliminary wear region and SAM refines the final segmentation.

## Pipeline Overview

The workflow is organized into three main stages.

### 1. SAM-based mask generation

The script masc_sam.py is used to generate segmentation masks with SAM. For each input image, the user manually defines a bounding box around the wear region. SAM then returns multiple candidate masks.

The script saves:

* the three candidate masks generated by SAM;
* overlay images for visual inspection;
* a consensus mask obtained by majority voting;
* a CSV file containing SAM scores and output information.

The majority voting strategy keeps a pixel as part of the wear region when at least two of the three SAM masks classify it as wear.

### 2. U-Net training

The script train_unet.py trains a U-Net model using the masks generated by SAM as reference masks.

The model uses:

* U-Net architecture from segmentation_models_pytorch;
* ResNet34 encoder;
* ImageNet pre-trained weights;
* binary segmentation output;
* BCEDiceLoss, combining Binary Cross-Entropy and Dice Loss;
* input image size of 256 × 256 pixels;
* batch size of 2;
* 50 training epochs;
* learning rate of 1e-4.

During training, the script evaluates the model using Dice Score and IoU Score on the validation set. The model with the best validation IoU is saved as unet_desgaste.pth.

### 3. Hybrid U-Net + SAM pipeline

The script unet_sam.py applies the complete hybrid pipeline to the test images.

The process follows these steps:

1. The trained U-Net predicts a preliminary probability map for the wear region.
2. A threshold is applied to convert the probability map into a binary mask.
3. The largest detected contour is used to generate a bounding box.
4. The bounding box is used as an automatic prompt for SAM.
5. SAM generates three candidate masks.
6. A majority voting strategy is applied to obtain the final refined mask.
7. The final masks, overlays, bounding box images, and comparison images are saved.

This approach aims to automate the generation of SAM prompts, reducing manual intervention in the segmentation process.

## Requirements

The main dependencies are:

opencv-python
numpy
torch
torchvision
segmentation-models-pytorch
segment-anything
tqdm

To install the required packages, use:

pip install opencv-python numpy torch torchvision segmentation-models-pytorch segment-anything tqdm

The SAM checkpoint must be downloaded separately and placed in the modelos/ folder:

modelos/sam_vit_h_4b8939.pth

## How to Run

### Step 1: Generate SAM masks

Run:

python scripts/masc_sam.py

By default, the script processes the following folders:

train/
valid/
test/

It is also possible to process only one folder by passing its name as an argument:

python scripts/masc_sam.py train

The generated masks are saved in:

mascaras/

The overlays are saved in:

overlays/

The SAM scores are saved in:

resultados/scores_sam.csv

### Step 2: Train the U-Net

Run:

python scripts/train_unet.py

The script uses the images from:

train/
valid/

and the corresponding masks from:

mascaras/train/
mascaras/valid/

The trained model is saved as:

modelos/unet_desgaste.pth

### Step 3: Run the hybrid U-Net + SAM pipeline

Run:

python scripts/unet_sam.py

The script processes the images from:

test/

The final predictions are saved in:

predicoes_sam_refinado/

This folder includes:

* U-Net preliminary masks;
* SAM-refined masks;
* overlay images;
* bounding box images;
* side-by-side comparison images.

## Notes

Before running the scripts, check and adjust the project paths if necessary. The scripts were developed for a local experimental setup and may require minor path adjustments depending on the repository structure.

The dataset images are not included in this repository. Users should download or prepare the image dataset separately and organize it according to the expected folder structure.

The SAM checkpoint is also not included and must be downloaded separately according to the original Segment Anything Model instructions.

## Methodological Considerations

The U-Net model was trained using masks derived from SAM-based segmentation. Therefore, the hybrid pipeline may incorporate part of the errors or tendencies present in the initial SAM masks. The main goal of the hybrid approach is not necessarily to outperform SAM with manually defined prompts, but to automate the generation of the input prompt used by SAM and reduce manual intervention during segmentation.

## Outputs

The pipeline generates binary segmentation masks and visual overlays that can be used for qualitative inspection and quantitative evaluation. These outputs can be compared with reference masks using metrics such as:

* Intersection over Union (IoU);
* Precision;
* Recall;
* absolute percentage difference in segmented area;
* absolute percentage difference in flank wear width (VB).
